CSS Flex

There is so much to learn about Flex that it would be difficult (impossible really) to cover all the configurations in one lesson. This lesson wil go over bringing elements inline, in a row, while occupying equal space; a common set up for desktop navigation.

If you see something wrong or vague, please holla at me. :)

Setup

First thing: you'll need at least one (flex) parent and child.

Think of flex as a box (the parent) that contains a single or multiple elements (child/children). The parent sets the rules and children follow the rules. Children can and sometimes do break the rules but for the sake of this lesson, we'll only look obedient children...

Flex Parent Rules

Consider the following setup:

This is a flex parent: 100% (max-width: 600px) x 250px

This is a single flex child.

To accomplish this set up, we start with declaring flex for the parent. This is done by assing the css rule: display: flex to the parent. This declares every immediate child to follow the flex rules. Keep in mind this only sets the rules for the immediate children, not nested elements. That is, if you have text, images, forms, etc., nested within the flex-child, the nested elements are not affected directly by the flex parent rules. However the contents can be used to determine the dimensions of the child. We'll go deeper in detail at a later time when we have a good understanding of the basics...

Next, we need to "tell" the flex-child to center vetically and horizontally. This is done by using align-items and justify-content.

align-items will affect the vertical layout of the child elements and we'll use justify-content (in this case) to affect the horizontal layout. For each, we asign the parameter of center. So far, the parent CSS should like like the following:

.flex-parent {
  display: flex;
  justify-content: center;
  align-items: center;
}

This is where I find the beauty in flex; these same rules will apply to all the children elements. So if I have 1, or 3 or 5 elements, they all adhere to the same rules without having to make changes to the css, making responsive layouts much easier to create.

The css has not changed from our original setup. We are still using the same flex parent rules and yet the child elements all behave as expected; they're centered neatly in a row and all in line because they all still fit within the width of the parent container. But what happens when one of the child's contents are too long? Well this is where we have to assign rules to the child elements..

The default behavior of the flex parent is to allow child elements to occupy space based on the child element's own initial dimensions (width and height) and then "wrap" to a next line if it outgrows the space. This behavior isnt and issue for child elements that "fit" but for elements like long menu options, you'll want to keep everything inline, or prevent the wrapping behavior by setting the rule: flex-wrap: nowrap

Note: Be Aware Of Your Availble Space!

Nowrap doesn't "force" elements to stay within the flex parent's dimensions. If the flex children do not "fit", they could get placed outside the parent container. Keep in mind that many factors are used to determine the length of the child element such as flex-basis, padding, word-spacing, font-size, width, margin... the list goes on.

Now, you're css code should look like this:

.flex-parent {
  display: flex;
  flex-wrap: nowrap;
  justify-content: center;
  align-items: center;
}

With the child elements in line, it looks better but not perfect. While the nowrap rule positions the child elements to all share the same space along its axis, some occupy more space because of the nested content. How can we fix the layout so that each child occupies the same amount of space horizontally? And then how can we fix the layout so that each child has the same height?

Growing and Shrinking - It All Starts Somewhere

For the next steps, we'll take a look at the parent rules first...

Keep in mind the declared height of the parent is 80 view-widths but no higher than 300px. We know that all the elemnts should be the same height, even if some have to stretch. So to accomplish this, we look back at the align-items rule. To keep things simple, we'll hide the text for now...

If you recall, the rule align-items affects the vertical alignment of the child elements. We need them all to stretch to fill the height of the parent container. To do that, we change the rule align-items: center to align-items: stretch.

Cool. Ok so now that we have all the child elements occupying the same vertical space, what about giving each child an equal share of horizontal space? For this we look at flex-grow, flex-shrink and assign the base of the width using flex-basis.

Flex-Grow

For flex-grow, we can assign 0 (the default value) or a positive number. (We can also assign initial or inherit but we'll discuss those at a future time.)

You can think of flex-grow like asking the child element "How much do you want to grow?" Well for this instance, we want to allow growth if there is space available so, we'll assign this to 1. If we did not want the child element(s) to grow, flex-grow would be 0.

Flex-Shrink

flex-shrink works almost identical to flex-grow in that you assign 0 or a positive number. The difference is that instead of growing, you're telling the child to shrink.

For our instance, we will assign 1 (default value) to our child elements, making each of the elements to shrink by 1 if the need arises. This is important when the viewport shrinks down to mobile. If we leave our children to only grow, they would get kicked out of the parent container when space decreases. Assigning a positive shrinking number will instruct the children elements to shrink if needed.

This number is relative to each child. This means I can assign 1 to Frederick but 3 to Virginia. This would result in Virginia shrinking 3 times while Frederick would only shrink by 1.

Flex-Basis

This is your child element's initial size. Think of flex-basis as the child's starting dimensions. This could be a length unit, precentage of the parent container or auto (default value), which is equal to the length of flex item. If the the length is not specified, the contents will be used to determine the length.

For this instance, we'll assign a length of 100px to the flex-basis. Since we're forcing all the child elements to grow and shrink with the same parameters based on 100px, they will all occupy the same horizontal space; click on the button to show the results...

So far, your code should reflect the following:

.flex-parent {
  display: flex;
  justify-content: center;
  align-items: center;
  flex-wrap: nowrap;
  align-items: stretch;
}
.flex-child {
  flex-grow: 1;
  flex-shrink: 1;
  flex-basis: 100px;
}

The .flex-child rules can be short-handed by:

.flex-child {
  flex-grow: 1 1 100px;
}

Now the last thing we need to do is vertically center the text...

Centering Vertically and Horizontally

Recall that a flex parent's rules effect the immediate nested element(s). In the examples above, the flex parent is wrapped around flex child, which is then wrappeed around a p element; the flex parent rules only effect the flex child, not the paragraph elements. So to center the paragraph content vertically, we have to assign flex parent rules to each of the flex children, essentially creating parents of children.

Note: Elements Get Kicked Off

We discussed earlier:
"Nowrap doesn't "force" elements to stay within the flex parent's dimensions. If the flex children do not "fit", they could get placed outside the parent container."
This is a perfect example of how additional styles can affect the layout of the child elements.

Oh no! What happened?

CSS happened. Our original .flex-parent class contained additional rules such as position, margin, width and height rules that also applied to the child elements when we applied the .flex-parent class. While it is encouraged to reuse css styles (is there a name for this?), my suggestion is to separate flex css from other styles. This can be done by creating a a class solely for flex layouts. For the example above, we'll create a separate .new-flex-parent class and another class called .group for anything other than flex.
